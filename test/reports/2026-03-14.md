## 本次改动

### skill/acp-client.sh — SSE 解析性能优化
- 流式/Card 模式：每行 SSE 事件 fork 2-3 次 jq → 整个流单进程 jq 处理
- Card 模式 tool 名称提取：sed fork → bash 内置字符串操作
- 新增 `_sse_data_lines()` 公共函数
- 预估：200 个 SSE 事件从 ~600 次 fork 降到 3 次

### main.py — lifespan 修复
- `app.router.lifespan_context = lifespan` 会覆盖 SDK 的 internal_lifespan
- 改为嵌套：`async with _original_lifespan(application)` 保留 SDK 的 ThreadPoolExecutor 初始化

### acp_sdk/server/app.py — Python 3.14 兼容 (vendor patch)
- `nonlocal executor` 在 Python 3.14 下报 NameError（变量仅有类型注解无赋值）
- 改为 `_state: dict = {}` 容器模式绕过闭包限制

### test/test_sse_parse.sh — 新增
- 18 个离线单元测试，覆盖流式/Card jq 表达式、SSE 行过滤、bash 字符串解析、端到端模拟
